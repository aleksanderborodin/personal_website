LLMs are trained to guess.

2025-09-08 3 min read Прочитать по-русски →

LLMs hallucinate — they confidently produce wrong answers. They almost never say "I don't know," even when they could. You might think this is a pretraining issue (bad data, underfitting, insufficient examples). In practice, it isn't.

OpenAI published research whose core finding collapses to one sentence: LLMs are trained to play a guessing game. During post-training, answers get graded for correctness, with no distinction drawn between "I don't know" and a confidently wrong reply. So the model learns to always produce the most plausible-looking answer — sometimes it gets lucky. Like any human taking a multiple-choice test, except the model doesn't know whether it's being evaluated or serving a real user.

Given this, you can add a post-training stage that penalizes "I don't know" less harshly. You can spin up several confidence-thresholded variants of an existing model by changing only post-training — no retraining from scratch. You can retrofit this onto already-shipped LLMs far cheaper than training a new one. Clean, actionable finding.

This hints at a broader pattern. Response length might work the same way: it's easier to be correct when the answer is short, and the model learns the minimum verbosity required to pass RL evals — not a bit more, even when the user would want more. You could reward models during RL not only for correctness but for how close their verbosity is to optimal for the question. Other attributes likely deserve the same treatment.

In classical ML terms:

Accuracy is a bad metric for LLMs.

This is only the base case. The paper itself has many more experimental observations on how post-training subtly breaks the model.
