Agents don't need MCP. They need CLI.

2026-03-22 2 min read Прочитать по-русски →

MCP was everywhere a year ago. Most people still haven't noticed the simple thing: an agent with access to a shell can use any CLI tool just as well.

Plain CLIs have obvious advantages:

  • Easier to build
  • Transparent and understandable in operation
  • Usable by both agents and humans
  • Thousands already exist for every task

Google figured this out — their new tool is a CLI, not another MCP server. Released only this month, clearly aimed at agents, but still a CLI.

Hard numbers on the comparison are thin. Some say MCP needs more context on simple tasks and less on complex ones, if the tools are well-wrapped and discoverable. Others say the difference is negligible, and when it isn't, it's because the CLIs weren't adapted for context efficiency. What everyone agrees on: CLI gives the agent the same tools at the same success rate, with far less operational overhead.

In my projects I'm using CLI.
