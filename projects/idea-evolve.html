Evolutionary multi-agent framework

Idea Evolve

Specialized Claude agents evolve solutions to hard optimization problems in parallel.

Problem

Why single-agent LLM prompting stalls on hard problems

Hard optimization problems resist single-shot LLM prompting. A single agent gets stuck in a local optimum, repeats the same ideas, and stops improving. The solution space is too large for one perspective to cover.

Best-of-N sampling from one model helps a little, but the samples are drawn from the same distribution — you get variance, not diversity. The failure mode is recognisable: after a few tries the model cycles through the same three approaches, never finding the fourth.

Approach

Evolutionary search with role-specialized agents

Idea Evolve runs a population of specialized agents in parallel, each producing candidate solutions. The best survive, get combined, and get mutated — a classic evolutionary loop, but with each generation's variation produced by agents with different roles instead of random mutation.

Each role contributes a different kind of variation: architects restructure, explorers branch sideways, researchers inspect what's missing. That gives the population genuine diversity across generations — the thing vanilla best-of-N can't deliver.

Agent roles

Three specialized Claude agents, one population

Architects
Propose high-level structure and problem decompositions.
Explorers
Probe alternative paths — cheap, diverse, willing to fail.
Researchers
Analyze intermediate results and surface what the rest of the system is missing.
Evolutionary loop

The generation cycle in pseudocode

At its core, Idea Evolve is a straightforward evolutionary loop. What's different is what produces the next generation's variation — specialized agents rather than random mutations or crossover operators.

# One generation of Idea Evolve
def evolve(problem, evaluate, population, generations=N):
    for g in range(generations):
        proposals = []
        # 1. Fan out in parallel across specialized roles
        proposals += spawn(role=Architect, seeds=population.top(k))
        proposals += spawn(role=Explorer,  seeds=population.sample(k))
        proposals += spawn(role=Researcher, seeds=population.top(k))

        # 2. Score every candidate with the user-provided fitness fn
        scored = [(evaluate(p), p) for p in proposals]

        # 3. Select survivors, feed back as seed context next generation
        population = select_top(scored + population.items, size=pop_size)

    return population.best()

How it works

Using it on your own problem

A human describes the problem and an evaluation function. Idea Evolve spawns an initial agent work session, records the solutions each role produces, scores them, and feeds the best back as seed context for the next generation. Useful for algorithmic search, prompt optimization, and any task where "try a lot, keep what works" is the right move.

The only thing the user needs to supply is the evaluation function. Everything else — generating, selecting, seeding, stopping — is handled by the loop. Fitness can be anything returning a scalar: unit-test pass rate, benchmark score, latency, a human-rated grade, or a composite.

When to reach for it

Good fits and poor fits

Good fits
  • Hard combinatorial or algorithmic search with a cheap, automatic scorer
  • Prompt or pipeline optimization where you can measure output quality
  • Creative problems where diversity of approach matters more than a single strong sample
  • Iterative refactoring tasks with a clear objective function
Poor fits
  • Problems with no automatic evaluator — human review blows up the cost/latency
  • One-shot queries a single strong model already solves
  • Latency-critical online paths — each generation is a round-trip to Claude
  • Tasks where the cost of an extra generation exceeds the expected quality gain
